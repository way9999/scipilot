# 完整问题库（Full Mode - 12 Questions）

## L1｜硬约束层（Context Constraints）

> 不问清楚，后面全白干

### Q1：研究阶段与要求

**目的**：确定深度下限

**提问**：
```
请告诉我你的研究阶段：
- [ ] 博士
- [ ] 硕士
- [ ] 本科

开题报告有哪些硬性要求？
- 页数要求：___
- 必须章节：___
- 偏向：理论 / 应用 / 均可
```

**映射**：文献量、理论深度、问题抽象程度

---

### Q2：学科与一级方向

**目的**：限定知识域

**提问**：
```
你的学科背景：
- 一级学科（如：计算机科学、物理学、经济学）：___
- 二级方向（如：机器学习、图学习、理论计算机）：___
```

**映射**：背景与研究现状的叙述风格

---

## L2｜问题空间层（Problem Space）

> 整个开题的"坐标原点"

### Q3：中心研究对象

**目的**：锁定研究核心

**提问**：
```
你目前的"中心研究对象"是什么？（不是方法，不是算法，而是对象）

参考选项：
- 某类数据结构（图 / 超图 / 张量 / 序列）
- 某类问题（划分 / 聚类 / 学习 / 推断 / 优化）
- 某类系统（推荐系统 / 网络 / 生物系统 / 社交网络）
- 其他：___
```

**映射**：论文题目、背景第一段

---

### Q4：研究类型偏向

**目的**：定位研究风格

> ⚠️ **领域适配**：根据 Q2 学科方向，从 [domain-adapters.md](domain-adapters.md) 加载对应选项

**默认提问**（计算机）：
```
你更偏向哪一类研究？（单选）
- [ ] 理论方法（证明、界、复杂度分析）
- [ ] 算法设计（新算法、优化现有算法）
- [ ] 应用驱动（解决实际问题、系统实现）
- [ ] 理论 + 实验混合
```

**映射**：研究意义的侧重点

---

## L3｜方法锚点层（Method Anchor）

> 避免"方向发散"的关键

### Q5：方法视角或工具

**目的**：锚定技术路线

> ⚠️ **领域适配**：根据 Q2 学科方向，从 [domain-adapters.md](domain-adapters.md) 加载对应选项

**默认提问**（计算机）：
```
你是否已有明确的方法视角或工具？（可多选，可空）
- [ ] 信息论（熵、互信息、编码）
- [ ] 优化（凸优化、组合优化、启发式）
- [ ] 概率模型（贝叶斯、图模型、采样）
- [ ] 深度学习（GNN、Transformer、生成模型）
- [ ] 几何/代数方法（流形、谱方法、张量分解）
- [ ] 其他：___
- [ ] 还没想好（完全没关系！）
```

**映射**：拟解决问题的技术路线

---

### Q6：方法来源

**目的**：理解创新点定位

**提问**：
```
这个方法思路的来源是？
- [ ] 已在文献中见过，想系统化/改进
- [ ] 自己的直觉/兴趣驱动
- [ ] 导师建议的方向
- [ ] 其他：___
```

**映射**：创新点叙述策略

---

## L4｜动机与痛点层（Motivation）

> "研究意义"的素材库

### Q7：兴趣来源

**目的**：挖掘原始动机

**提问**：
```
你为什么对这个方向感兴趣？（非学术语言完全可以）
- [ ] 遇到过实际工程问题
- [ ] 理论上有"不爽"的地方
- [ ] 现有方法太 heuristic / 不够优雅
- [ ] 结果不可解释 / 缺乏理论保证
- [ ] 纯粹觉得有意思
- [ ] 其他：___
```

**映射**：选题背景、问题提出段

---

### Q8：现有方法的不满

**目的**：定位改进方向

**提问**：
```
你觉得现有方法"最不舒服"的地方是？（可多选）
- [ ] 缺乏统一理论框架
- [ ] 目标函数没有物理/信息意义
- [ ] 参数太多，难以调优
- [ ] 难以扩展到大规模
- [ ] 缺乏可解释性
- [ ] 实验结果不稳定
- [ ] 其他：___
```

**映射**：研究现状中的"不足分析"

---

## L5｜可行性层（Feasibility）

> 导师最关心的部分

### Q9：现有资源

**目的**：评估可行性

**提问**：
```
你目前具备哪些资源？（可多选）
- [ ] 数据（有现成数据集 / 可获取数据）
- [ ] 算法基础（熟悉相关算法）
- [ ] 代码基础（有可复用代码 / 熟悉相关工具）
- [ ] 理论基础（相关数学/理论背景）
- [ ] 计算资源（GPU / 集群）
- [ ] 领域知识（了解应用场景）
```

**映射**：可行性分析、技术路线可信度

---

### Q10：成果偏向

**目的**：明确产出预期

**提问**：
```
你希望这个工作最终偏向？
- [ ] 理论论文（定理、证明、界）
- [ ] 实验论文（方法 + 大量实验验证）
- [ ] 应用论文（系统实现、案例研究）
- [ ] 混合型
```

**映射**：预期成果

---

## L6｜约束与偏好层（Soft Constraints）

> 提高输出质量，非硬性必须

### Q11：导师偏好与禁忌

**目的**：避免方向性错误

**提问**：
```
导师有没有明确偏好或禁忌？（可选填）
- 偏好：___
- 禁忌：___

常见示例：
- 不要纯深度学习
- 一定要有理论证明
- 必须有真实数据实验
- 要发顶会/顶刊
```

**映射**：全局约束

---

### Q12：交互风格偏好

**目的**：调整后续引导方式

**提问**：
```
在后续的开题报告撰写中，你更希望我：
- [ ] 强引导（像导师一样，主动给建议和框架）
- [ ] 弱引导（你来主导，我提供支持和反馈）
```

**映射**：SKILL 后续交互风格
