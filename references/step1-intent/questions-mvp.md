# 精简问题库（MVP Mode - 5 Questions）

> 最小可用问答版本，足以生成：开题报告大纲、初版研究动机、可行技术路线草图

## Q1：研究阶段 + 学科

**整合自**：L1-Q1, L1-Q2

**提问**：
```
请简要介绍你的基本情况：
1. 研究阶段：博士 / 硕士 / 本科
2. 一级学科：___
3. 研究方向：___
```

**映射**：深度下限、知识域、叙述风格

---

## Q2：中心研究对象

**整合自**：L2-Q3

**提问**：
```
你的"中心研究对象"是什么？

举例：
- 数据结构类：图、超图、张量、时序数据
- 问题类：划分、聚类、推荐、优化
- 系统类：推荐系统、社交网络、知识图谱

你的回答：___
```

**映射**：论文题目、背景第一段

---

## Q3：偏理论 / 偏应用

**整合自**：L2-Q4, L5-Q10

> ⚠️ **领域适配**：根据 Q1 学科，从 [domain-adapters.md](domain-adapters.md) 加载对应选项

**默认提问**（计算机）：
```
你的研究更偏向？
- [ ] 理论（证明、界、复杂度）
- [ ] 算法（设计新方法、优化性能）
- [ ] 应用（解决实际问题、系统实现）
- [ ] 混合
```

**映射**：研究意义侧重、预期成果类型

---

## Q4：是否已有方法锚点

**整合自**：L3-Q5, L3-Q6

> ⚠️ **领域适配**：可根据 Q1 学科，从 [domain-adapters.md](domain-adapters.md) 提供领域相关的方法示例作为引导

**提问**：
```
你目前是否已有想法或方法方向？

- [ ] 有，具体是：___
- [ ] 有一些模糊想法：___
- [ ] 还没有（完全没关系，我们一起探索）
```

**映射**：技术路线、创新点方向

---

## Q5：当前最不满意的点

**整合自**：L4-Q7, L4-Q8

**提问**：
```
你觉得现有工作"最让你不满意"或"最想改进"的是什么？

可以是：
- 理论不完善
- 方法不够优雅
- 效果不好
- 不可解释
- 无法扩展
- 或者任何让你觉得"应该能做得更好"的地方

你的回答：___
```

**映射**：选题背景、问题提出、研究现状不足分析

---

```

  │
  ▼

  │
  ▼
┌─────────────────────────────────────────┐
│  领域适配：加载 domain-adapters.md       │
│  匹配学科 → 替换 Q3/Q4 选项              │
└─────────────────────────────────────────┘
  │
  ▼
Q2: 研究对象 ──────────────────► 锁定核心概念
  │
  ▼
Q3: 研究类型 ──[适配后选项]───► 明确产出方向
  │
  ▼

  │
  ▼
[信息采集完成] ──► 进入 Phase 2
```

## 追问策略

MVP 模式下，若某问答案过于模糊，可针对性追问：

| 模糊答案 | 追问方向 |
|----------|----------|
| Q2 不明确 | "能举一个你最近读过的、与你方向相关的论文吗？" |
| Q4 完全没有 | "有没有让你印象深刻的方法或技术？" |
| Q5 说不清 | "如果让你吐槽现有方法，你会说什么？" |
