# Step 3: 结构冻结 (Structure Freeze)

> 在明确研究方向与约束条件的前提下，自动构建结构清晰、逻辑自洽的开题报告总体框架，作为后续内容扩展与调整的骨架。

## 目标

将 Step 1（用户意图）+ Step 2（事实知识库）整合，生成**可编辑的开题报告大纲**，并通过分层交互确认后冻结。

---

## 前置条件

- Step 1 已完成（意图采集）
- Step 2 已完成（知识库覆盖度达标）

---

## 核心流程

```
1. 加载学科大纲模板 → outline-adapters.md
2. 生成一级大纲 → 用户确认/调整
3. 展开二级大纲 → 用户确认/调整
4. 素材映射 → 文献分配到章节
5. 写入大纲文件 → outline.md
6. 冻结确认 → 进入Step 4
```

---

## Step 3.1: 加载学科大纲模板

根据 Step 1 的 Q2（学科方向）加载对应模板：

```
function loadOutlineTemplate(discipline):
    if discipline in OUTLINE_ADAPTERS:
        return OUTLINE_ADAPTERS[discipline]
    else:
        return GENERIC_TEMPLATE
```

各学科模板定义见 [outline-adapters.md](outline-adapters.md)

---

## Step 3.2: 一级大纲生成与确认

**生成逻辑：**

基于模板 + 用户意图，生成一级标题列表：

```markdown
## 开题报告大纲（一级）

1. [必写] 选题背景与意义
2. [必写] 国内外研究现状
3. [必写] 研究内容与目标
4. [必写] 研究方法与技术路线
5. [可选] 可行性分析
6. [必写] 预期成果与创新点
7. [可选] 进度安排
```

**用户交互：**

```
请确认一级大纲结构：
- [确认] 按当前结构继续
- [调整] 合并/拆分/重排章节
- [增删] 添加或删除章节
```

**调整示例：**
- 用户："把可行性分析改为必写"
- 用户："在第3章后增加'关键问题分析'"
- 用户："删除进度安排"

---

## Step 3.3: 二级大纲展开与确认

一级确认后，逐章展开二级标题：

**示例（计算机学科）：**

```markdown
## 2. 国内外研究现状
   2.1 [必写] 领域发展脉络（约300字）
   2.2 [必写] 代表性方法综述（约500字）
   2.3 [可选] 现有方法局限性分析（约200字）
   2.4 [必写] 本研究的切入点（约200字）
```

**用户交互（逐章）：**

```
第2章「国内外研究现状」的二级结构：
- [确认] 按当前结构
- [调整] 修改子节
- [跳过] 暂不展开，后续补充
```

---

## Step 3.4: 素材映射

将 Step 2 知识库中的文献分配到各章节：

**映射规则：**

| 文献标签 | 映射目标章节 |
|----------|--------------|
| 背景文献 | 1.选题背景、2.研究现状 |
| 方法参考 | 4.技术路线、5.可行性 |
| 问题佐证 | 1.选题背景、3.研究内容 |

**映射输出示例：**

```markdown
### 素材映射

| 章节 | 关联文献 |
|------|----------|
| 1. 选题背景 | @Wang2023_survey, @Li2024_analysis |
| 2. 研究现状 | @Wang2023_survey, @Zhang2023_method |
| 4. 技术路线 | @Zhang2023_method, @Chen2024_framework |
```

---

## Step 3.5: 写入大纲文件

生成 `outline.md` 文件，包含完整结构信息：

**文件路径：** `research-scaffold/outline.md`

**文件格式：**

```markdown
---
status: frozen
created: {timestamp}
discipline: {学科}
last_confirmed: {timestamp}
---

# 开题报告大纲

## 元信息
- **研究题目（暂定）**: {从Step 1提取}
- **学科方向**: {Q2}
- **总篇幅建议**: {根据学科调整}

---

## 章节结构

### 1. 选题背景与意义 [必写]
**篇幅**: 800-1200字
**关联文献**: @Wang2023_survey, @Li2024_analysis
**要点提示**:
- 领域大背景（宏观）
- 具体问题引出（微观）
- 研究意义（理论+实践）

#### 1.1 研究背景（约400字）
#### 1.2 研究意义（约400字）

---

### 2. 国内外研究现状 [必写]
**篇幅**: 1500-2000字
**关联文献**: @Wang2023_survey, @Zhang2023_method, ...

...
```

---

## Step 3.6: 冻结确认

**最终确认交互：**

```
大纲已生成并保存至 outline.md

请选择：
- [冻结] 确认大纲，进入Step 4（渐进式生成）
- [修改] 直接编辑outline.md后重新确认
- [重来] 重新生成大纲
```

**冻结后约束：**
- `outline.md` 头部 `status` 改为 `frozen`
- Step 4 将严格按照冻结大纲执行
- 如需修改，需显式"解冻"后重新确认

## 篇幅建议参考

### 按学科调整

| 学科 | 总篇幅 | 重点章节 |
|------|--------|----------|
| 计算机/AI | 8000-12000字 | 技术路线(30%)、研究现状(25%) |
| 材料/化学 | 10000-15000字 | 实验方案(30%)、研究现状(25%) |
| 生物医学 | 10000-15000字 | 研究方法(30%)、背景意义(20%) |
| 经管 | 8000-12000字 | 理论框架(25%)、研究设计(25%) |
| 通用 | 8000-12000字 | 均衡分布 |

### 各章节篇幅比例（通用）

| 章节 | 比例 | 字数参考(万字报告) |
|------|------|-------------------|
| 选题背景与意义 | 10-15% | 800-1500字 |
| 国内外研究现状 | 20-30% | 2000-3000字 |
| 研究内容与目标 | 15-20% | 1500-2000字 |
| 研究方法/技术路线 | 20-30% | 2000-3000字 |
| 可行性分析 | 5-10% | 500-1000字 |
| 预期成果与创新点 | 10-15% | 1000-1500字 |
| 进度安排 | 5% | 300-500字 |

---

## 异常处理

| 情况 | 处理策略 |
|------|----------|
| Step 2文献不足 | 提示用户补充，或标记章节为"待补充素材" |
| 用户多次调整 | 记录调整历史，提供"回退"选项 |
| 学科未匹配 | 使用通用模板，提示用户自定义 |
| 用户跳过确认 | 保存当前状态，允许后续继续 |

---

## 与Step 4衔接

`outline.md` 作为 Step 4 的**唯一输入**：
- Step 4 按章节顺序逐个生成内容
- 每章生成时读取：篇幅要求、关联文献、要点提示
- 用户确认后更新章节状态为 `completed`
