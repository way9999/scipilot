# 研究写作系统升级手册

> 将外部研究写作经验收敛为本仓库可执行的论文工作流资产。

## 设计原则

- **计划驱动**：先写 `paper-plan.md`，把文献、实验、图表、章节拆成可检查任务。
- **证据优先**：没有文献摘录或实验结果的论断不进入终稿。
- **章节冻结**：方法与实验先稳定，引言与摘要最后收敛。
- **双轨输出**：同时维护 `drafts/*.md` 与 `output/latex/*`，方便快速编辑和最终投稿。

## 关键资产

- `drafts/paper-draft.md`：当前正文草稿
- `drafts/paper-outline.md`：章节骨架与证据槽位
- `drafts/paper-plan.md`：issue-driven 执行计划
- `drafts/paper-revision-prompts.md`：润色、补引用、LaTeX 转写提示包
- `output/latex/paper.tex`：LaTeX 初稿
- `output/latex/references.bib`：当前参考文献条目

## 推荐循环

1. 用 `generate-paper-draft` 生成论文套件。
2. 先补 `knowledge-base/` 中缺失的核心文献，再回填证据笔记。
3. 用 `paper-plan.md` 推进图表、实验与章节。
4. 用 `paper-revision-prompts.md` 做学术润色、补引用和内审。
5. 将成熟段落迁移到 `output/latex/paper.tex`，最终统一参考文献格式。

## 提示词策略

### 学术润色（消除 AI 痕迹）

5 步执行顺序：

1. **禁用表达扫描**：逐段检查中英文禁用表达（见 `writing-style-adapters.md` 通用质量规则），标记所有命中项
2. **冗余压缩**：删除冗余动词结构（"进行了研究"→"研究了"）、多余连接词（"此外...另外"不连续使用）
3. **句式去重**：检测连续相同句式开头，改写为多样化表达
4. **语气校准**：保留学术严谨语气，去除过度自信（"clearly"、"obviously"）和过度谦虚（"仅仅是初步尝试"）
5. **终检**：通读全文确认无 AI 痕迹残留，句长节奏自然（长短交替）

### 补引用润色

4 步执行顺序：

1. **无引用断言扫描**：标记所有事实性陈述（非常识）中缺少引用的位置
2. **文献匹配**：从 `knowledge-base/` 中检索可支撑该论断的文献，优先选择高相关性、高质量来源
3. **引用嵌入**：将引用自然融入行文，避免机械堆砌（"[1][2][3][4]"→ 分散到不同论点）
4. **完整性校验**：确认所有正文引用在参考文献列表中存在，所有参考文献在正文中被引用

### LaTeX 转写

5 步执行顺序：

1. **结构映射**：将 Markdown 标题层级映射为 `\section`/`\subsection`/`\subsubsection`
2. **列表散文化**：将项目符号列表改写为连贯段落（Methods 中的步骤列表除外，转为 `enumerate`）
3. **公式转换**：行内公式用 `$...$`，独立公式用 `\begin{equation}...\end{equation}` 并编号
4. **图表转换**：Markdown 表格转 `\begin{table}`，图片转 `\begin{figure}`，确保 caption 和 label 完整
5. **引用转换**：`[@AuthorYear]` 转为 `\cite{AuthorYear}`，生成对应 `.bib` 条目

### 内审检查

4 维度结构化报告：

1. **内容维度**：论点是否有充分证据支撑？是否有逻辑跳跃？贡献是否与 Gap 对应？
2. **写作维度**：是否有 AI 痕迹？句式是否多样？段落长度是否均衡？过渡是否自然？
3. **格式维度**：引用格式是否统一？图表编号是否连续？缩写是否首次定义？
4. **可复现性维度**：实验设置是否完整？数据集是否可获取？代码是否可运行？超参数是否列全？

每个维度输出格式：

```markdown
#### [维度名称]

| # | 问题 | 位置 | 风险等级 | 修改建议 |
|---|------|------|----------|----------|
| 1 | ... | §3.2 第2段 | 高/中/低 | ... |
```

## 提交前终检清单

论文提交前逐项确认：

### 内容完整性（5 项）

- [ ] 所有章节已完成，无 `[TODO]` 或 `[待补充]` 占位符
- [ ] Abstract 中的数值与正文/表格一致
- [ ] 所有贡献点在正文中有对应的实验验证
- [ ] Discussion 回应了 Introduction 中提出的研究问题
- [ ] Conclusion 不引入新信息，仅总结已有内容

### 写作质量（5 项）

- [ ] 全文无 AI 痕迹禁用表达（通过禁用表达列表扫描）
- [ ] 限定词密度 ≤3 次/千字（"显著"、"重要"、"关键"等）
- [ ] 无项目符号列表（Methods 除外）
- [ ] 句长节奏自然，无连续相同句式
- [ ] 所有模糊表达已替换为具体数值/场景

### 格式规范（4 项）

- [ ] 所有图表在正文中被引用，编号连续
- [ ] 所有缩写首次出现时有完整定义
- [ ] 引用格式统一，参考文献列表完整
- [ ] 符合目标期刊/会议的格式模板要求

### 可复现性（5 项）

- [ ] 数据集来源、版本、划分方式已说明
- [ ] 实验环境已说明（硬件、软件版本）
- [ ] 超参数完整列出（学习率、batch size、epoch 等）
- [ ] 随机种子已固定并报告
- [ ] 代码/数据可用性声明已包含

## 常见拒稿原因对照

| # | 高频拒稿原因 | 预防措施 |
|---|-------------|----------|
| 1 | 贡献不清晰 / 新颖性不足 | Introduction P4 明确列出贡献点，与现有工作做差异化对比 |
| 2 | 实验不充分 / 缺少基线对比 | 至少 3 个基线方法，含最新 SOTA；消融实验验证各组件贡献 |
| 3 | 写作质量差 / AI 痕迹明显 | 执行学术润色流程，通过禁用表达扫描 |
| 4 | 相关工作覆盖不足 | Related Work 覆盖近 3 年核心文献，经 `/lit-verify` 验证 |
| 5 | 统计分析缺失 | 报告 p 值/置信区间/效应量，多次运行报告均值±标准差 |
| 6 | 可复现性差 | 完成"可复现性"终检清单全部 5 项 |
| 7 | 图表质量低 | 矢量图格式（PDF/SVG），字号≥8pt，配色色盲友好 |
| 8 | 格式不符合要求 | 提交前对照目标期刊 Author Guidelines 逐项检查 |
