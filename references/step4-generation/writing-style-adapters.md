# 领域写作风格适配器 (Writing Style Adapters)

> 根据学科方向自动适配写作风格、术语规范与行文习惯

## 使用方式

1. 根据 Q2（学科方向）加载对应风格配置
2. 生成内容时遵循该领域的写作规范
3. 若学科未列出，使用"通用学术"风格

---

## 通用质量规则（所有学科生效）

> 以下规则优先级高于学科专项规则，在所有写作输出中强制执行。

### 句长与节奏

- 英文：平均句长 15-20 词，单句不超过 40 词
- 中文：平均句长 30-50 字，单句不超过 80 字
- 长短句交替，避免连续 3 句以上句长相近（差异 <20%）

### 中文禁用表达

| 禁用表达 | 替换建议 | 原因 |
|----------|----------|------|
| 众所周知 | [删除，直接陈述事实并加引用] | 无引用断言 |
| 不言而喻 | [删除，改为明确论证] | 回避论证 |
| 值得注意的是 | [删除，直接陈述] | AI 痕迹 |
| 近年来...受到广泛关注 | 具体说明哪些年、哪些团队、什么进展 | 模糊+AI 痕迹 |
| 进行了深入研究 | 研究了 / 分析了 / 探讨了 | 冗余动词结构 |
| 取得了显著成果 | 具体说明什么成果、什么指标 | 模糊断言 |
| 具有重要意义 | 说明对谁重要、为什么重要 | 空洞评价 |

### 英文禁用表达

| 禁用表达 | 替换建议 | 原因 |
|----------|----------|------|
| It is worth noting that | [删除，直接陈述] | AI 痕迹 |
| In recent years | Since 2020 / Over the past decade（具体化） | AI 痕迹+模糊 |
| novel (自称) | we propose / we present | 自我评价应由审稿人判断 |
| clearly / obviously | [删除，或提供证据] | 回避论证 |
| significantly (无统计检验) | 给出 p 值或效应量 | 滥用统计术语 |
| delve / delve into | examine / investigate / analyze | AI 痕迹 |
| game-changer / groundbreaking | [删除，用数据说明影响] | 夸大 |
| a plethora of | many / numerous / several | AI 痕迹 |
| in order to | to | 冗余 |
| it should be noted that | [删除，直接陈述] | AI 痕迹 |

### 冗余短语清理

**中文：**
| 冗余表达 | 精简为 |
|----------|--------|
| 进行了研究 | 研究了 |
| 作出了分析 | 分析了 |
| 对...进行了探讨 | 探讨了... |
| 给予了充分的考虑 | 考虑了 |
| 做出了详细的描述 | 描述了 |
| 实现了有效的提升 | 提升了 |

**英文：**
| 冗余表达 | 精简为 |
|----------|--------|
| conduct an investigation of | investigate |
| perform an analysis of | analyze |
| is able to | can |
| due to the fact that | because |
| in the event that | if |
| at the present time | now / currently |

### 模糊→具体转换

| 模糊表达 | 要求 |
|----------|------|
| 显著提升/significantly improved | 必须跟具体数值（如 "提升了 12.3%"） |
| 广泛应用/widely used | 必须举≥2个具体应用场景 |
| 大量数据/large dataset | 必须给出具体数量（如 "包含 50,000 条样本"） |
| 较好的性能/good performance | 必须给出指标名称和数值 |
| 近年来/in recent years | 必须给出具体年份范围 |

### 列表使用限制

- Methods / 技术路线 / 实验设置章节：允许使用编号列表和项目符号
- 其他所有章节（Introduction、Results、Discussion、Conclusion 等）：禁止项目符号列表，必须改写为连贯散文段落
- 表格不受此限制

## 计算机科学 / 人工智能

### 写作风格

| 维度 | 规范 |
|------|------|
| 整体基调 | 客观严谨，技术导向 |
| 人称使用 | 第一人称复数（"本文"、"我们"）或被动语态 |
| 时态 | 现在时为主，描述已有工作用过去时 |
| 段落长度 | 中等（150-250字/段） |

### 术语规范

| 场景 | 正确用法 | 避免用法 |
|------|----------|----------|
| 方法描述 | "提出"、"设计"、"构建" | "发明"、"创造" |
| 效果描述 | "提升"、"改进"、"优化" | "大幅提高"、"显著增强"（需数据支撑） |
| 问题描述 | "存在局限"、"面临挑战" | "很差"、"不行" |
| 对比描述 | "相比于"、"与...不同" | "比...好很多" |

### 常用句式

```markdown
# 引出问题
近年来，随着...的发展，...问题日益突出。
现有方法在...方面存在以下局限：

# 方法介绍
针对上述问题，本文提出了一种基于...的方法。
具体而言，本文的主要贡献包括：

# 实验描述
为验证所提方法的有效性，本文在...数据集上进行了实验。
实验结果表明，所提方法在...指标上达到了...

# 总结展望
本文提出的方法在...方面取得了一定进展，但仍存在...的局限。
未来工作将从...方向进一步研究。
```

### 引用格式
- 数字标注：[1], [2,3], [4-7]
- 文中提及：文献[1]、Wang等[2]

### 禁忌

- ❌ 过度口语化："这个方法很牛"
- ❌ 无依据断言："本方法是最好的"
- ❌ 模糊表述："大概"、"可能"、"也许"（需量化）

## 材料科学 / 材料工程

### 写作风格

| 维度 | 规范 |
|------|------|
| 整体基调 | 严谨客观，注重实验细节 |
| 人称使用 | 被动语态为主，偶用"本文" |
| 时态 | 方法描述用过去时，普遍规律用现在时 |
| 段落长度 | 较长（200-300字/段） |

### 术语规范

| 场景 | 正确用法 | 避免用法 |
|------|----------|----------|
| 制备描述 | "合成"、"制备"、"沉积" | "做"、"弄" |
| 性能描述 | "表现出"、"呈现"、"具有" | "有"、"是" |
| 机理描述 | "归因于"、"可能源于" | "因为"、"所以" |
| 数据描述 | "测得"、"观察到"、"检测到" | "发现"（谨慎使用） |

### 常用句式

```markdown
# 背景引入
XX材料因其优异的...性能，在...领域具有广泛的应用前景。
然而，现有XX材料普遍存在...的问题，限制了其实际应用。

# 方法描述
采用...方法制备了...样品。具体步骤如下：
将...溶解于...中，在...条件下反应...小时。

# 结果描述
图X为...的XRD图谱。从图中可以看出，...
表X列出了不同条件下样品的...性能。结果表明，...

# 机理分析
上述性能的提升可归因于...
这一现象可能与...有关。
```

### 引用格式
- 上标数字：^[1]^, ^[2,3]^
- 或数字标注：[1], [2]

### 禁忌

- ❌ 缺乏实验条件细节
- ❌ 定性描述无定量支撑："性能显著提高"
- ❌ 机理解释过于绝对

## 生物学 / 生命科学

### 写作风格

| 维度 | 规范 |
|------|------|
| 整体基调 | 严谨科学，逻辑清晰 |
| 人称使用 | 被动语态为主 |
| 时态 | 方法/结果用过去时，已知事实用现在时 |
| 段落长度 | 中等偏长（180-280字/段） |

### 术语规范

| 场景 | 正确用法 | 避免用法 |
|------|----------|----------|
| 基因/蛋白 | 斜体基因名，正体蛋白名 | 格式混乱 |
| 数据描述 | "检测到"、"鉴定出"、"筛选得到" | "找到"、"看到" |
| 结论表述 | "表明"、"提示"、"暗示" | "证明"（谨慎使用） |
| 机制描述 | "可能通过...途径"、"推测..." | 过度绝对化 |

### 常用句式

```markdown
# 背景引入
XX是...的关键调控因子，在...过程中发挥重要作用。
研究表明，XX通过...途径参与...的调控。

# 方法描述
为探究XX的功能，本研究构建了...模型。
采用...技术检测了...的表达水平。

# 结果描述
如图X所示，与对照组相比，实验组中...
Western blot结果显示，XX的表达水平显著上升（P<0.05）。

# 讨论
上述结果提示，XX可能通过调控...参与...过程。
这一发现与既往研究[1]的报道一致/不一致。
```

### 引用格式
- 作者-年份：(Wang et al., 2023)
- 文中提及：Wang等(2023)

### 禁忌

- ❌ 缺乏统计分析
- ❌ 过度解读实验结果
- ❌ 基因/蛋白命名不规范

### 专项格式规则

| 规则 | 说明 | 示例 |
|------|------|------|
| 基因名斜体 | 基因名称使用斜体 | *TP53*、*BRCA1* |
| 蛋白名正体 | 蛋白质名称使用正体 | p53、BRCA1 蛋白 |
| 物种二名法 | 首次出现用完整拉丁名（斜体），后续可缩写 | *Escherichia coli* → *E. coli* |
| 统计报告格式 | 包含检验方法、统计量、自由度、p 值 | t(24) = 3.45, p = 0.002 |
| Person-first language | 描述患者时使用 person-first 表述 | "patients with diabetes"（非 "diabetic patients"） |

## 能源与动力工程

### 写作风格

| 维度 | 规范 |
|------|------|
| 整体基调 | 工程导向，注重实用性 |
| 人称使用 | "本文"、被动语态交替 |
| 时态 | 描述方法用过去时，规律用现在时 |
| 段落长度 | 中等（150-250字/段） |

### 术语规范

| 场景 | 正确用法 | 避免用法 |
|------|----------|----------|
| 效率描述 | "热效率"、"㶲效率"、"能量利用率" | "效率"（需明确类型） |
| 参数描述 | 带单位的准确数值 | 模糊数值 |
| 优化描述 | "优化"、"改进"、"提升" | "改良"（偏口语） |
| 仿真描述 | "数值模拟"、"仿真计算" | "模拟"（需明确方法） |

### 常用句式

```markdown
# 背景引入
随着能源需求的增长和环保要求的提高，...技术受到广泛关注。
传统...系统存在...效率低、...损失大等问题。

# 方法描述
本文建立了...的数学模型，采用...方法进行求解。
数值模拟采用...软件，网格数量为...，边界条件设置如下...

# 结果描述
图X给出了...参数对...性能的影响规律。
当...从...增加到...时，系统效率从...%提升至...%。

# 工程意义
该方法可使系统热效率提高X个百分点，年节能量约为...。
```

### 引用格式
- 数字标注：[1], [2-5]

### 禁忌

- ❌ 仿真参数不明确
- ❌ 数值无单位
- ❌ 忽略工程可行性分析

## 物理学

### 写作风格

| 维度 | 规范 |
|------|------|
| 整体基调 | 逻辑严密，数学化表述 |
| 人称使用 | "我们"（理论物理常用）或被动语态 |
| 时态 | 定理/定律用现在时，推导过程用过去时 |
| 段落长度 | 偏短（100-200字/段），穿插公式 |

### 术语规范

| 场景 | 正确用法 | 避免用法 |
|------|----------|----------|
| 理论描述 | "哈密顿量"、"拉格朗日量" | 非标准译名 |
| 推导过程 | "由...可得"、"代入...得到" | "所以"、"然后" |
| 近似描述 | "在...近似下"、"忽略...项" | 隐含近似不说明 |
| 结果描述 | "与理论预期一致"、"偏差在...量级" | 定性描述 |

### 常用句式

```markdown
# 理论推导
考虑一个...系统，其哈密顿量可写为：
H = ...
在...近似下，上式可简化为...

# 数值计算
采用...方法对上述方程进行数值求解。
计算参数设置如下：...

# 结果分析
图X展示了...随...的变化关系。
计算结果与文献[1]的实验数据吻合良好，最大偏差约为...%。
```

### 引用格式
- 数字标注：[1], [2,3]
- 部分领域用作者-年份

### 禁忌

- ❌ 公式无编号
- ❌ 符号未定义先使用
- ❌ 近似条件不说明

### 专项格式规则

| 规则 | 说明 | 示例 |
|------|------|------|
| SI 单位 | 所有物理量使用 SI 单位，数值与单位间留空格 | 9.8 m·s⁻² |
| 公式编号 | 所有被引用的公式必须编号 | 见式(1)、由式(3)可得 |
| 符号首次定义 | 每个物理符号首次出现时必须定义 | 其中 $k_B$ 为玻尔兹曼常数 |
| 矢量/标量标注 | 矢量用粗体或箭头，标量用斜体 | **F** = m**a** 或 $\vec{F} = m\vec{a}$ |
| 不确定度报告 | 实验数据需报告不确定度 | (3.14 ± 0.02) × 10⁸ m·s⁻¹ |

## 化学 / 化学工程

### 写作风格

| 维度 | 规范 |
|------|------|
| 整体基调 | 严谨规范，注重细节 |
| 人称使用 | 被动语态为主 |
| 时态 | 实验描述用过去时 |
| 段落长度 | 中等偏长（200-300字/段） |

### 术语规范

| 场景 | 正确用法 | 避免用法 |
|------|----------|----------|
| 化合物命名 | IUPAC命名或通用名 | 非标准名称 |
| 反应描述 | "合成"、"制备"、"转化" | "做"、"得到" |
| 表征描述 | "采用...表征"、"通过...分析" | "用...看" |
| 产率描述 | "收率为X%"、"转化率达X%" | "产量不错" |

### 常用句式

```markdown
# 实验部分
将...（X g, X mmol）溶于...（X mL）中，加入...
在...℃下搅拌反应X小时后，经...处理得到目标产物。
收率：X%；熔点：X℃；^1H NMR (CDCl3, 400 MHz): δ ...

# 结果讨论
图X为产物的红外光谱图。位于...cm^-1处的吸收峰归属于...
与原料相比，产物在...处出现新峰，表明...反应已完成。

# 机理分析
推测该反应经历以下历程：首先...发生...，随后...
```

### 引用格式
- 上标数字：^[1]^, ^[2]^

### 禁忌

- ❌ 试剂规格不明
- ❌ 反应条件缺失
- ❌ 表征数据不完整

### 专项格式规则

| 规则 | 说明 | 示例 |
|------|------|------|
| IUPAC 命名 | 化合物首次出现使用 IUPAC 系统命名 | 2-丙醇（异丙醇） |
| SI 单位 | 所有物理量使用 SI 单位 | kJ·mol⁻¹（非 kcal/mol） |
| 光谱数据格式 | NMR/IR/MS 数据按期刊规范格式报告 | ¹H NMR (400 MHz, CDCl₃): δ 7.26 (s, 1H) |
| CCDC 编号 | 晶体结构需提供 CCDC 存档编号 | CCDC 2345678 |
| 反应条件完整性 | 温度、时间、溶剂、催化剂、当量均需明确 | 80°C, 12 h, DMF, Pd(PPh₃)₄ (5 mol%) |

## 经济学 / 管理学

### 写作风格

| 维度 | 规范 |
|------|------|
| 整体基调 | 逻辑清晰，论证充分 |
| 人称使用 | "本文"、"本研究" |
| 时态 | 理论用现在时，实证用过去时 |
| 段落长度 | 较长（250-350字/段） |

### 术语规范

| 场景 | 正确用法 | 避免用法 |
|------|----------|----------|
| 假设提出 | "假设"、"命题" | "猜测"、"推断" |
| 变量描述 | "自变量"、"因变量"、"控制变量" | "X"、"Y"（需定义） |
| 结果表述 | "显著正相关"、"显著负向影响" | "有关系"、"有影响" |
| 稳健性 | "稳健性检验"、"内生性处理" | 忽略稳健性 |

### 常用句式

```markdown
# 理论分析
基于...理论，XX与YY之间存在如下关系：
当...时，企业倾向于...，从而导致...

# 假设提出
综上分析，本文提出以下假设：
H1：XX对YY具有显著正向影响。

# 实证方法
为检验上述假设，本文构建如下计量模型：
Y = α + β₁X + γControls + ε
其中，Y为...，X为...，Controls包括...

# 结果解读
表X报告了基准回归结果。列(1)显示，XX的系数为...（p<0.01），
表明XX每增加一个单位，YY将提高...%，假设H1得到支持。
```

### 引用格式
- APA格式：(Wang & Li, 2023)
- 文中提及：Wang和Li (2023)

### 禁忌

- ❌ 无理论支撑的假设
- ❌ 忽略内生性问题
- ❌ 显著性解读错误

## 通用学术风格（学科未匹配时）

### 写作风格

| 维度 | 规范 |
|------|------|
| 整体基调 | 客观严谨 |
| 人称使用 | "本文"、被动语态 |
| 时态 | 视具体内容选择 |
| 段落长度 | 中等（150-250字/段） |

### 通用原则

1. **客观性**：避免主观评价，用数据说话
2. **准确性**：术语准确，数据精确
3. **逻辑性**：论点-论据-结论结构清晰
4. **规范性**：格式统一，引用规范

## 风格加载函数

```python
def load_style(discipline):
    STYLES = {
        "计算机": CS_STYLE,
        "人工智能": CS_STYLE,
        "材料": MATERIALS_STYLE,
        "生物": BIO_STYLE,
        "能动": ENERGY_STYLE,
        "物理": PHYSICS_STYLE,
        "化学": CHEMISTRY_STYLE,
        "经济": ECON_STYLE,
        "管理": ECON_STYLE,
    }

    for key, style in STYLES.items():
        if key in discipline:
            return style

    return GENERIC_STYLE
```
